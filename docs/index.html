문제를 이렇게 풀었어요

01
BDNS

실시간 채팅 메시지 누락·순서 꼬임

문제
팬카페 실시간 채팅에서 동시 접속이 늘면 메시지가 간헐적으로 누락되거나 표시 순서가 뒤바뀌었습니다.
원인
WebSocket 연결·재연결 시 세션 관리가 일관되지 않았고, 클라이언트가 도착 순서대로 메시지를 정렬하지 않았습니다.
해결
STOMP 기반으로 구독 채널과 세션을 정리하고, 서버에서 발행하는 메시지에 순번·타임스탬프를 부여해 클라이언트가 순서대로 렌더링하도록 했습니다.
결과
동시 접속 상황에서도 메시지 누락 없이 순서가 보장되어 채팅이 안정화됐습니다.
02
BDNS

게시글 좋아요·조회수 동시성 정합성

문제
인기 게시글에 좋아요·조회 요청이 몰리면 카운트가 실제보다 적게 집계되는 문제가 있었습니다.
원인
조회 후 +1 후 저장하는 read-modify-write 방식이라 동시 요청 사이에 갱신이 덮어써졌습니다(lost update).
해결
카운트 갱신을 단일 UPDATE 증감 쿼리로 바꾸고, 중복 반응은 유니크 제약으로 막아 한 번만 반영되도록 처리했습니다.
결과
동시 트래픽에서도 좋아요·조회수가 정확히 집계되어 통계 신뢰도가 올라갔습니다.
03
BDNS

금칙어 필터 우회 표현 차단

문제
채팅·게시글에서 금칙어 사이에 공백이나 특수문자를 끼워 넣어 필터를 우회하는 경우가 있었습니다.
원인
단순 포함(contains) 매칭만 사용해 변형된 표현을 걸러내지 못했습니다.
해결
입력을 정규화(공백·특수문자·반복 문자 제거)한 뒤 금칙어를 매칭하고, 자모 분리 변형까지 검사하도록 필터를 개선했습니다.
결과
우회 표현까지 차단되어 커뮤니티 운영·모니터링 부담이 줄었습니다.
04
도어키친

공휴일을 제외한 배송일 자동 계산

문제
구독 배송일을 자동 생성할 때 공휴일·주말에도 배송일이 잡혀 운영자가 매번 수동으로 조정해야 했습니다.
원인
단순히 주기(N일)만 더해 날짜를 계산했고, 공휴일 정보를 반영하지 않았습니다.
해결
공공 공휴일 API를 연동해 공휴일을 캐싱하고, 스케줄러가 배송일 계산 시 공휴일·주말을 건너뛰도록 로직을 보강했습니다.
결과
운영자의 수기 조정이 사라지고, 배송 일정이 자동으로 정확하게 생성됐습니다.
05
도어키친

주소 → 배송구역 자동 매핑 실패

문제
회원 주소를 배송구역에 자동 매핑할 때 일부 주소가 매칭되지 않아 배송 가능 여부 판단이 어긋났습니다.
원인
도로명·지번 주소 표기가 제각각이고, 일부 신주소가 구역과 바로 매칭되지 않았습니다.
해결
SGIS 지오코딩으로 주소를 좌표로 변환해 배송구역과 매칭하고, 실패 건은 따로 모아 운영자가 수동 보정할 수 있게 했습니다.
결과
자동 매핑률이 올라가고, 매칭 실패 건도 누락 없이 관리할 수 있게 됐습니다.
06
도어키친

아임웹 페이지 삽입 위젯의 로딩·도메인 이슈

문제
아임웹 결제·마이페이지에 삽입한 위젯 스크립트가 일부 환경에서 로드되지 않거나 API 호출이 차단됐습니다.
원인
외부 도메인(아임웹)에서 우리 API를 호출하면서 CORS 정책에 막혔고, 위젯 번들 경로·캐시 문제가 겹쳤습니다.
해결
위젯 전용 API에 허용 도메인 CORS를 명시하고, 위젯을 독립 번들로 빌드해 안정된 경로에서 서빙하도록 배포 구조를 정리했습니다.
결과
아임웹 외부 페이지에서도 위젯이 안정적으로 동작하며 배송지역 체크가 정상화됐습니다.
07
도어키친

아임웹 OAuth 토큰 만료·동시 갱신 경쟁

문제
아임웹 API 호출 중 액세스 토큰이 만료되면 호출이 실패하고, 여러 요청이 동시에 토큰을 갱신하려다 충돌했습니다.
원인
토큰 만료를 사전에 확인하지 않았고, 갱신 로직에 동시성 제어가 없었습니다.
해결
만료 전 선제 갱신하도록 하고, 갱신이 한 번만 일어나도록 제어해 리프레시 토큰을 안전하게 회전시켰습니다.
결과
토큰 만료로 인한 API 실패가 사라지고 아임웹 연동이 안정적으로 유지됐습니다.
08
도어키친

아임웹 주문 동기화 중복·누락

문제
아임웹 webhook으로 주문을 받아 동기화할 때 같은 주문이 중복 저장되거나 일부가 누락됐습니다.
원인
webhook이 재전송될 수 있는데 멱등 처리가 없었고, 스케줄러 동기화와 시점이 겹치면서 충돌이 났습니다.
해결
주문 외부 ID를 유니크 키로 두어 upsert 처리하고, 스케줄러가 마지막 동기화 시각 이후 건만 보정하도록 재동기화 로직을 분리했습니다.
결과
중복·누락이 사라지고 아임웹 주문 데이터가 일관되게 유지됐습니다.
09
IMWEB LMS

대용량 강의 영상 업로드 실패·재시도

문제
강사가 큰 강의 영상을 올릴 때 네트워크가 끊기면 처음부터 다시 업로드해야 했습니다.
원인
단일 요청으로 전체 파일을 전송해, 중간 실패 시 이어서 올릴 방법이 없었습니다.

결과
대용량 업로드 실패율이 크게 줄고, 강사의 업로드 경험이 개선됐습니다.
